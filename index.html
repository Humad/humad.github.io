Motivate Bot

Motivate Bot is an application that sends motivational and/or wholesome text messages to its subscribers at regular intervals.
The thought that triggered the creation of this application was, ‚ÄúI want to make people smile‚Äù. The idea is simple: send a message to the subscribers. However, I had to put together lots of different components in code to make this work.

First, there had to be an API that could provide random quotes. Luckily, I built one years ago.

Second, I needed to connect to Twilio‚Äôs API to be able to send text messages.

And finally, I needed to have my own database of users to go through at a regular interval and determine whether or not to send a message to them (depending on whether or not they were still subscribed, and how much time had passed since they received their last message). I also gave users the ability to select the frequency of text messages for themselves.



Initially, Motivate Bot was supposed to be a Facebook messenger application but Facebook took too long to approve my app request, so I switched over to text messages. When first creating Motivate Bot, I wrote a Medium article about how to create Facebook Messenger bots.

Motivate Bot is currently live. There is also an iOS app in development for me to track app usage.

Created using JavaScript, Node.js, Twilio, MongoDB, Heroku.

Reel Feels

A content sharing platform that provides insights into user emotions and emotional changes. I collaborated with 3 other developers on this project.

The idea is to have a platform where content creators, similar to those on YouTube or Twitch, can either upload or share their content for users to view. Apart from the usual metrics such as user views and likes/dislikes, the content creators will be able to see (either in graph form or via some other visual representation) how the users felt while viewing content. This is extremely valuable information for content creators as it allows them to determine weaknesses in their content and equips them with the data necessary to produce better content.

The application will require a device with a front facing camera, such as a laptop or a phone, in order to read the user‚Äôs facial features using the Affectiva API to determine their emotions. The Affectiva API runs purely on the browser and does not require any additional resources.



Reel Feels is currently offline because hosting it is currently too expensive and it is no longer actively maintained. However, feel free to download the code and run it on your computer!

Created using: Python, JavaScript, SQL, Django, Affectiva, Google Charts.

Movie Guide Voice Assistant

When smart speakers were still new technology, they didn't have as many features as they did today. When I first bought the Google Home, it couldn't answer questions about movies very well. Instead of waiting for better features, I decided to create my own.

Movie Guide offers information to the user about movies. Users can interact with it using their voice or through text.

Movie Guide is an agent created using API.AI with a webhook that was created using Node.js and Express.js. It is integrated with Actions on Google and can be used on devices such as smartphones running Android, and Google Home.



Since there aren't a lot of good movie information APIs (surprisingly), I had to create a little web scraper using Cheerio that would go through IMDB and gather some information from there.

Smart speakers have evolved since I created Movie Guide. Since it was no longer needed, I decided to take it down. However, if you like, you can download the code and run it locally!

Created using: JavaScript, Node.js, API.AI, Express
